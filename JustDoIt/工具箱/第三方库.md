## 包管理工具

工程使用了哪种包管理工具，主要通过查看项目根目录下的**特征配置文件**来确定。不同包管理工具会生成特定的配置文件，以下是常见场景的判断方法：
### 一、通用判断方法：查看项目根目录的配置文件

打开你的工程文件夹，查看是否存在以下特征文件（按平台 / 语言分类）：

#### 1. **C/C++ 项目**

- **vcpkg**：存在 `vcpkg.json`（清单模式）或 `vcpkg/` 目录（传统模式），可能还有 `vcpkg_installed/`（安装的依赖目录）。
- **Hunter**：项目的 `CMakeLists.txt` 中会包含 `HunterGate()` 初始化代码，且可能有 `~/.hunter/` 缓存目录（全局）。
- **Conan**：存在 `conanfile.txt` 或 `conanfile.py`，以及 `conan.lock`（依赖锁定文件）。
- **pkg-config**：存在 `*.pc` 文件（库的配置描述），但它更像依赖查询工具，不算严格的包管理器。

#### 2. **iOS/macOS（Objective-C/Swift）项目**

- **CocoaPods**：存在 `Podfile` 和 `Podfile.lock`，以及 `Pods/` 目录（存放下载的依赖）。
- **Carthage**：存在 `Cartfile` 和 `Cartfile.resolved`，以及 `Carthage/` 目录（编译后的框架）。
- **Swift Package Manager (SPM)**：存在 `Package.swift`（项目依赖配置），且 Xcode 项目中可能显示 “Swift Packages” 依赖。

#### 3. **JavaScript/TypeScript 项目**

- **npm**：存在 `package.json` 和 `package-lock.json`（npm 5+ 自动生成）。
- **Yarn**：存在 `package.json` 和 `yarn.lock`，可能还有 `.yarn/` 目录（Yarn 2+）。
- **pnpm**：存在 `package.json` 和 `pnpm-lock.yaml`，以及 `node_modules/.pnpm/` 目录（依赖存储路径）。

#### 4. **Python 项目**

- **pip**：存在 `requirements.txt`（依赖列表），但 pip 本身不严格管理版本，通常配合 `pip-tools` 生成 `requirements.txt`。
- **Poetry**：存在 `pyproject.toml` 和 `poetry.lock`，以及 `poetry.toml`（配置文件）。
- **pipenv**：存在 `Pipfile` 和 `Pipfile.lock`。

#### 5. **Java/Kotlin 项目**

- **Maven**：存在 `pom.xml`（项目对象模型文件）。
- **Gradle**：存在 `build.gradle` 或 `build.gradle.kts`（Kotlin 脚本），以及 `gradle/` 目录。

#### 6. **Rust 项目**

- **Cargo**：存在 `Cargo.toml`（依赖配置）和 `Cargo.lock`（版本锁定）。

#### 7. **Go 项目**

- **Go Modules**：存在 `go.mod`（依赖声明）和 `go.sum`（校验和文件），这是 Go 1.11+ 内置的包管理方式。

### 二、其他辅助判断方法

1. **查看构建脚本或说明文档**项目的 `README.md` 或 `BUILD.md` 中通常会说明依赖管理工具（如 “使用 `npm install` 安装依赖” 或 “运行 `vcpkg install` 下载库”）。
    
2. **检查项目的构建命令**
    
    - 若常用 `npm run build` 或 `yarn build` → 对应 npm/Yarn。
    - 若用 `cmake --build` 且有 `vcpkg` 相关参数 → vcpkg。
    - 若用 `pod install` → CocoaPods。
3. **查看依赖安装目录**
    
    - `node_modules/` → npm/Yarn/pnpm。
    - `Pods/` → CocoaPods。
    - `vcpkg_installed/` → vcpkg。

### 三、特殊情况：多个包管理工具共存

有些项目可能混合使用多种工具（如前端项目同时有 `package.json` 和 `CMakeLists.txt`，分别管理 JS 和 C++ 依赖），此时需根据**具体模块**判断：

- 前端代码部分 → 看 `package.json` 等 JS 相关文件。
- 原生代码部分 → 看 `vcpkg.json` 或 `CMakeLists.txt` 中的包管理配置。
