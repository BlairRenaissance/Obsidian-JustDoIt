
在函数声明中，变量是否有名字是可选的，有也会被编译器忽略。 

在函数定义中, 变量是否有名字也是可选的，如果函数内用不到的话最好写成没名字的，就不会触发警告或错误。

那么为什么用不到还要保留呢？这种情况主要出现在以下几种典型场景中：

## 函数迭代升级的兼容性处理

``` cpp
// 旧版本函数
void processData(int data) { /* 使用data参数 */ }

// 新版本函数（不再需要data参数）
void processData(int /*unused*/) { /* 新实现 */ }
```

- 保留参数避免破坏现有调用代码
- 特别适用于库函数维护场景
- 注释/`*unused*/`是常见的标记方式

---
## 运算符重载的语法区分

``` cpp
class Digit {
public:
    Digit& operator++();    // 前缀形式（无参数）
    Digit operator++(int);  // 后缀形式（哑元参数）
                            // 参数仅用于语法区分，实际不参与运算
};
```

- 后缀运算符的int参数是语言标准要求的语法标记
- 该参数永远不被使用，仅作为编译器识别标志

---
## 模板元编程的类型萃取

``` cpp
template<typename T>
void func(T /*类型标签*/) {
    // 通过类型特征而非参数值进行逻辑分支
    if constexpr(std::is_integral_v<T>) {...}
}
```

- 参数类型在编译期参与决策
- 实际运行时参数值可能完全不被使用

---

> [!Tip] Best practice
> When a function parameter exists but is not used in the body of the function, do not give it a name. You can optionally put a name inside a comment.
