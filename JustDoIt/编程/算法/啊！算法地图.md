# 任务进度统计

```dataviewjs
// 安全地获取当前文件的所有任务
const allTasks = dv.pages().file.tasks
    .filter(t => t?.path === dv.current()?.file?.path) || [];

// 添加整体进度统计
const totalTasks = allTasks.length;
const totalCompleted = allTasks.filter(t => t.completed).length;
const totalPercent = totalTasks > 0 ? Math.round((totalCompleted / totalTasks) * 100) : 0;

dv.el("div", `
<p> 总完成率: <strong>${totalPercent}%</strong></p>
<p>✅ 已完成: ${totalCompleted} 个 | 📥 未完成: ${totalTasks - totalCompleted} 个 | 📊 总计: ${totalTasks} 个</p>
<progress value="${totalCompleted}" max="${totalTasks}" style="width:100%; height:20px;"></progress>
`);
```

## 8.2 二叉搜索树

二叉搜索树（BST）是二叉树的一种特殊表示形式，它满足如下特性：
- 每个节点中的值必须大于（或等于）存储在其左侧子树中的任何值。
- 每个节点中的值必须小于（或等于）存储在其右子树中的任何值。

### 树的遍历

对于二叉搜索树，我们可以通过**中序遍历**得到一个**递增的**有序序列。因此，中序遍历是二叉搜索树中最常用的遍历方法。

- [ ] LC . 验证二叉搜索树
- [ ] LC . 二叉搜索树迭代器

### 基础操作

二叉搜索树的有优点是，即便在最坏的情况下，也允许你在O(n)的时间复杂度内执行所有的搜索、插入、删除操作。

通常来说，如果你想==有序==地存储数据或者需要同时执行搜索、插入、删除等多步操作，二叉搜索树这个数据结构是一个很好的选择。

- [ ] LC . 二叉搜索树中的插入操作

- [ ] LC . 数据流中的第K大元素
- [ ] LC . 二叉搜索树的最近公共祖先
- [ ] LC . 存在重复元素 III

### 平衡二叉搜索树

一个高度平衡的二叉搜索树（平衡二叉搜索树）是在插入和删除任何节点之后，可以自动保持其高度最小。也就是说，有 N 个节点的平衡二叉搜索树，它的高度是 logN 。并且，每个节点的两个子树的高度不会相差超过 1。
